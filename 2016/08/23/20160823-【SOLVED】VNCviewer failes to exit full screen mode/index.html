【SOLVED】VNCviewer failes to exit full screen mode

使用VNCviewer连接服务器可以使用服务器的图形界面，对于需要图形界面的应用来说非常重要，比如我处理数据时要用R画图，必须要有图形界面支持。

我桌面机使用windows 10,用VNCviewer连接服务器的centOS 7.0,但是VNCvewer在进入全屏界面之后无法再退出，原本可以使用F8唤出VNCviewer工具界面的也不管用了。这种时候我如果在桌面机前是可以使用ctrl + alt + del唤出windows任务管理器结束VNCviewer的进程的。

但是。。。有一种情况是我回宿舍了，然后用宿舍的机器通过mstsc远程桌面连接到桌面机，这时，会直接进到VNCviewer对应的centOS系统没有办法退出。我以前的做法是在服务器上kill掉VNC进程，这时远程桌面会给你退到windows的桌面。现在发现了一种更好的方式。

在还没有全屏时把VNCviewer的菜单打开，比如在标题栏上右击，点开选项。将输入栏的菜单键设置成非F8，我设置成了F11，这样，你在全屏时可以通过按F11唤出VNCviewer的菜单栏。可能是centOS的F8已经有用了，导致了这样一个bug吧。总之，换掉菜单键之后解决了问题。
